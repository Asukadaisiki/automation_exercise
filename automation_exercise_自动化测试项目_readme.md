# Automation Exercise 自动化测试实战

本项目以 **Automation Exercise** 官方提供的 API / UI 场景为被测对象，围绕 **测试理论、测试工具、测试用例编写** 三个高频面试点，构建了一套可落地、可回归的自动化测试实践，用于体现测试工程基础能力与自动化落地能力。

---

## 一、项目目标与定位

- 将常见测试理论（等价类、边界值、场景法等）**真正落地到测试用例与自动化代码中**，而非停留在概念层面
- 构建一套 **接口自动化为主、UI 冒烟为辅** 的自动化回归方案
- 形成从 **用例设计 → 自动执行 → 报告分析 → 持续回归** 的完整测试闭环

该项目可直接作为 **测试 / 测试开发实习** 的核心项目经验。

---

## 二、测试范围说明

### 1. 接口测试范围（核心）

基于 Automation Exercise 官方 API 文档，覆盖以下模块：

- 产品与品牌接口
  - 产品列表查询
  - 品牌列表查询

- 搜索接口
  - 正常关键词搜索
  - 非法/异常输入（空值、特殊字符、超长字符串）
  - 缺失参数校验

- 登录与鉴权校验
  - 正确/错误账号登录
  - 缺失参数异常返回

- 账户生命周期（端到端场景）
  - 创建账户 → 查询账户 → 更新账户 → 删除账户

接口测试重点覆盖：
- 正常流 / 异常流
- 参数校验与错误码
- 接口依赖与数据隔离

---

### 2. UI 自动化范围（冒烟）

基于官方 UI Test Cases，选取关键业务路径进行验证：

- 用户注册 / 登录主流程
- 产品列表与详情页访问
- 核心页面跳转与关键元素校验

UI 自动化仅作为 **端到端验证与加分项**，不追求数量，强调稳定性与可维护性。

---

## 三、测试理论落地说明

本项目并非单纯“写脚本”，而是将测试理论与实践一一对应：

- **等价类划分**：
  - 搜索关键词的合法 / 非法输入分类

- **边界值分析**：
  - 空值、极端长度、特殊字符

- **场景法**：
  - 账户完整生命周期端到端验证

- **异常流测试**：
  - 缺参、非法参数、错误方法请求

所有测试点均先体现在 **测试用例文档** 中，再映射到自动化脚本，保证可追溯性。

---

## 四、测试用例设计规范

### 用例统一模板

- 用例 ID / 标题
- 前置条件
- 测试步骤
- 测试数据
- 断言点（状态码 / 响应字段 / 业务规则）
- 预期结果
- 用例类型（冒烟 / 回归）

用例文档位于：
```
docs/testcases.md
```

自动化代码中的用例命名、Allure 标题与用例 ID 一一对应，保证用例与代码一致。

---

## 五、测试工具与技术选型

### 工具分工说明

- **Postman**
  - 接口探索性验证
  - 快速构造请求与联调
  - 维护集合与环境变量

- **Pytest + Requests**
  - 构建可维护的接口自动化回归
  - 参数化、数据驱动、fixture 前后置管理

- **Allure**
  - 测试结果可视化
  - 自动采集请求 / 响应 / 失败信息

- **Playwright**
  - UI 冒烟自动化
  - 失败截图与 trace 辅助定位

- **GitHub Actions**
  - 自动触发回归执行
  - 形成持续测试闭环

## 七、自动化与回归说明

- 接口自动化支持多次重复执行，数据互不污染
- 通过 fixture 管理测试数据创建与清理
- 自动生成 Allure 报告，辅助问题定位
- CI 中默认执行接口自动化，UI 冒烟可按需触发

---

## 八、可写入简历的项目描述

**Automation Exercise 自动化测试实战（个人项目）**

- 基于官方 API 场景，系统性设计接口测试用例，采用等价类、边界值与场景法拆分测试点，覆盖产品、搜索、登录校验及账户生命周期等核心业务。
- 使用 Pytest + Requests 构建接口自动化回归体系，通过参数化与 fixture 管理前后置与测试数据清理，解决接口依赖与数据污染问题。
- 对接口响应进行多维断言并生成 Allure 测试报告，失败场景自动采集请求/响应信息，降低问题定位与回归成本。
- 结合 Postman与 GitHub Actions，实现接口验证资产化与自动化回归闭环；补充 Playwright UI 冒烟用例验证端到端稳定性。
